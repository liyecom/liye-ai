# External Exposure Levels

> **Status**: CANONICAL
> **Purpose**: 定义系统对外暴露的等级和边界
> **Created**: 2025-12-31

---

## Overview

系统对外暴露采用分级制度，每个等级有明确的：
- 使用者范围
- 权限边界
- 进入条件
- 退出条件

```
严格程度: E0 > E1 > E2 > E3

E0 ──────────────────────────────────────────────── E3
│                                                    │
最严格                                              最开放
（当前）                                            （禁止直接进入）
```

---

## E0 – Internal Only

> **当前状态**: ✅ 激活

### 定义

仅限核心维护者使用，系统处于开发和验证阶段。

### 使用者

- 核心开发者
- 系统维护者
- 内部测试人员

### 权限

| 权限 | 状态 |
|------|------|
| 访问所有代码 | ✅ |
| 运行 Runner | ✅ |
| 修改 T1 机制 | ✅ |
| 查看所有 Case | ✅ |
| 修改配置 | ✅ |

### 边界

- 不对外展示
- 不对外描述
- 不发布任何演示

---

## E1 – Controlled Demo

> **当前状态**: 🔒 未开放

### 定义

受控演示模式，用于向受邀观察者展示"机制存在"。

### 使用者

- 受邀观察者
- 内部决策者
- 潜在合作评估者

### 权限

| 权限 | 状态 |
|------|------|
| 观看固定 Demo | ✅ |
| 查看特定 Case 结果 | ✅ |
| 修改输入参数 | ❌ |
| 选择 T1 机制 | ❌ |
| 运行 Runner | ❌ |
| 访问代码 | ❌ |

### 目的

```
✅ 展示"机制存在"
✅ 展示"治理框架"
✅ 展示"验证方法"

❌ 不承诺效果
❌ 不暗示自动化能力
❌ 不提供任何商业承诺
```

### Demo 限制

```yaml
demo_constraints:
  fixed_cases: [case_01, case_04, case_12]  # 仅限特定 Case
  fixed_domain: [ppc, listing]               # 仅限特定 Domain
  input_locked: true                         # 输入不可修改
  output_only_verdict: true                  # 仅展示 verdict
  no_mechanism_selection: true               # 不可选择机制
```

### 进入条件

| 条件 | 要求 |
|------|------|
| Usage Boundary 签署 | 必须 |
| 观察者身份验证 | 必须 |
| Demo 脚本预审 | 必须 |
| 无商业承诺 | 必须 |

## E2 – Partner / Research Access

> **当前状态**: 🔒 未开放

### 定义

合作方或研究用途访问，允许有限度的交互。

### 使用者

- 正式合作方
- 学术研究者
- 技术评估者

### 权限

| 权限 | 状态 |
|------|------|
| 选择 Domain | ✅ |
| 提供自定义 Context | ✅ |
| 查看 Verdict 详情 | ✅ |
| 选择 T1 机制 | ❌ |
| 修改 Runner 配置 | ❌ |
| 访问源代码 | ❌ |
| 自动执行 | ❌ |

### 强制要求

```
┌─────────────────────────────────────────────────────────┐
│ E2 强制签署文件                                          │
├─────────────────────────────────────────────────────────┤
│ 1. Usage Boundary Agreement                             │
│ 2. Non-Commercial Use Declaration (if research)         │
│ 3. Data Handling Agreement                              │
│ 4. No Effect Guarantee Acknowledgment                   │
└─────────────────────────────────────────────────────────┘
```

### 进入条件

| 条件 | 要求 |
|------|------|
| 完成 E1 周期 | ≥ 1 个周期无事故 |
| 合作意向书 | 必须 |
| Usage Boundary 签署 | 必须 |
| 用途声明 | 必须 |
| 退出条款同意 | 必须 |

### ⚠️ 重要限制

```
┌─────────────────────────────────────────────────────────┐
```

## Level Transition Rules

### 升级规则

```
E0 → E1: 治理文档完成 + E1 条件满足
E1 → E2: E1 无事故周期 × 1 + E2 条件满足
E2 → E3: E2 无事故周期 × 2 + Productization Gate
```

### 降级规则

```
任何级别 → E0: 发现误用 / 越权 / 效果承诺
E2 → E1: 合作方违反 Boundary
E1 → E0: Demo 脚本被滥用
```

### 事故定义

| 事故类型 | 严重程度 |
|----------|----------|
| 效果承诺 | 高 → 立即降级 |
| 越权访问 | 高 → 立即降级 |
| 商业滥用 | 高 → 立即降级 |
| 数据泄露 | 高 → 立即降级 |
| 误解传播 | 中 → 评估后决定 |
