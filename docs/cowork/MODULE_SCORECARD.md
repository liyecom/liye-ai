## 评分标准说明

### 维度定义

| 维度 | 说明 | 评分 |
|------|------|------|
| **使用强度** | 是否在当前主路径或近期活跃 | 1=几乎不用, 5=每日使用 |
| **杠杆价值** | 是否为多模块复用的基础设施 | 1=单点使用, 5=全局基础 |
| **不可替代性** | 是否存在成熟外部替代 | 1=可完全替代, 5=独有核心 |
| **维护成本** | 复杂度、耦合度、心智负担 | 1=极高成本, 5=几乎零维护 |
| **改动风险** | 是否牵一发而动全身 | 1=极高风险, 5=隔离良好 |

### 归类定义

| 归类 | 含义 |
|------|------|
| **Keep** | 保留并值得升级 |
| **Freeze** | 冻结，明确不再演进 |
| **Extract** | 适合拆出为 tools / 独立模块 |
| **Archive** | 应迁入 archive |
| **Delete（建议）** | 明确低价值，但绝不执行删除 |
