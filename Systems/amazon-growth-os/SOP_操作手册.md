## 二、准备工作

### 检查清单（开工前必查）

```
□ 1. 我有 Anthropic API Key（Claude 的密钥）
□ 2. 我有卖家精灵账号（可选，但推荐）
□ 3. 我知道要分析的产品/ASIN
□ 4. 电脑能上网
```

### 获取 API Key（如果没有）

**Anthropic API Key**（必须）：
1. 打开 https://console.anthropic.com/
2. 注册/登录
3. 点击 "API Keys" → "Create Key"
4. 复制保存（只显示一次！）

**卖家精灵 API Key**（可选）：
1. 登录卖家精灵后台
2. 找到 API 设置
3. 申请开通

### 首次设置（只做一次）

```bash
# 第 1 步：进入目录
cd ~/Documents/liye_workspace/LiYe_OS/Systems/amazon-growth-os

# 第 2 步：创建配置文件
cp .env.example .env

# 第 3 步：编辑配置文件，填入你的密钥
nano .env
# 或者用 VS Code 打开：code .env

# 第 4 步：填入内容（把 xxx 换成你的真实密钥）
ANTHROPIC_API_KEY=<YOUR_ANTHROPIC_API_KEY>

# 第 5 步：保存退出（nano 是 Ctrl+O 保存，Ctrl+X 退出）
```

## 三、场景 A：新品上架

> 📦 **我有个新产品，需要关键词和 Listing**

### 你需要准备

```
□ 产品名称或核心关键词（如 "yoga mat", "phone case"）
□ 目标市场（默认美国站）
```

### 操作步骤

```
┌──────────────────────────────────────────────────────────┐
│ 步骤 1：进入工作目录                                      │
└──────────────────────────────────────────────────────────┘

  cd ~/Documents/liye_workspace/LiYe_OS/Systems/amazon-growth-os

┌──────────────────────────────────────────────────────────┐
│ 步骤 2：运行新品分析命令                                  │
└──────────────────────────────────────────────────────────┘

  ./run.sh --mode launch --product "你的产品名"

  例如：
  ./run.sh --mode launch --product "Yoga Mat"
  ./run.sh --mode launch --product "Washable Runner Rug"

┌──────────────────────────────────────────────────────────┐
│ 步骤 3：等待（约 3-5 分钟）                               │
└──────────────────────────────────────────────────────────┘

  你会看到：
  - 🔍 正在分析关键词...
  - 📊 正在计算 TES 评分...
  - ✍️ 正在生成 Listing...
  - ✅ 完成！

┌──────────────────────────────────────────────────────────┐
│ 步骤 4：查看结果                                          │
└──────────────────────────────────────────────────────────┘

  结果保存在两个地方：

  1. 终端直接显示

  2. 文件保存：
     - optimized_listing.md （最新结果）
     - reports/markdown/Launch_产品名_时间戳.md （历史记录）
```

## 六、常见问题

### Q1：报错 "ANTHROPIC_API_KEY not set"

**原因**：没有配置 API 密钥

**解决**：
```bash
# 检查 .env 文件是否存在
ls -la .env

# 如果不存在，创建一个
cp .env.example .env

# 编辑并填入你的密钥
nano .env
```

---

### Q2：报错 "卖家精灵 API 调用失败"

**原因**：卖家精灵 API 没配置或配额用完

**解决方案 A**：手动导出数据
```
1. 登录卖家精灵网站
2. 搜索关键词/反查 ASIN
3. 导出 Excel 文件
4. 放到 data/inputs/ 目录
5. 运行时指定文件：
   ./run.sh --mode launch --product "Yoga Mat" --file_path "data/inputs/keywords.xlsx"
```

**解决方案 B**：配置 API Key
```bash
# 编辑 .env，添加：
SELLERSPRITE_API_KEY=你的密钥
```

---

### Q3：运行很慢 / 卡住了

**可能原因**：
1. 网络问题（API 调用超时）
2. 第一次运行，在下载模型

**解决**：
```bash
# 检查网络
ping api.anthropic.com

# 如果是第一次运行，耐心等待 5-10 分钟
```

---

### Q4：结果不满意 / 想重新生成

**解决**：
```bash
# 直接再运行一次，会覆盖之前的结果
./run.sh --mode launch --product "Yoga Mat"

# AI 每次生成会略有不同，多试几次选最好的
```

---

### Q5：想看历史记录

**解决**：
```bash
# 所有历史报告都在 reports/markdown/ 目录
ls reports/markdown/

# 用任何文本编辑器打开查看
cat reports/markdown/Launch_Yoga_Mat_20251222.md
```
