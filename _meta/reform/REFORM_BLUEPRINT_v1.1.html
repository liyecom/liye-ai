§1总判

骨架真实而优秀，皮肤大面积幻觉。AGE 的五层契约管线是实弹级的（最近一组 compile→verdict→report 全链产物落在 2026-07-02），fail-closed 写门链 G0–G7 逐行可核，审批已按 action_kind 三层分级，asin-growth v5 的"文档即合约"结构与 Anthropic 官方 SOTA 同构。但三层都挂着与现实脱节的自我描述：20 个 YAML agent 和 2 个 crew 没有任何执行路径（CrewAI 依赖根本不存在）；ads-governance 描述的 4 个核心脚本在 git 全史中从未存在；liye_os 的 SKILL_CONSTITUTION 自称 HARD BLOCK 而执法挂着 "(planned)" 五个月；真正扣扳机的 live 广告写入口 loamwise-ops 游离在 git 之外。

因此改革不是推倒重来——执行面值得保留和推广。改革是三件事：诚实化（让描述=现实，删掉规格僵尸与假治理面）、资产硬化（经验→手册→剧本→软件的梯度成为制度）、闭环通电（学习层 Layer B 目前是真空：live 库无 fact_measurement_verdict 表，verdict JSON 躺在 out/ 没进真值层）。

§2概念定版（已对齐，落盘为制度）

三件套互相咬合：业务主干说"做什么"，五概念说"用什么载体"，硬化梯度说"资产往哪沉"。

// 业务主干（你的手绘 + 闭环下半场）
数据 → 因子 → 决策 → 执行 → 回执 → 验证 → 学习
每个"如何XX?" = 一个 Skill compile→execute→receipt→verdict（已实弹） ← Layer B 真空，D9 通电
横切：治理门禁（G0–G7 · 灰度 · 人审检查点）　│　结果回灌：因子再加权 · 技能再进化
概念量化类比本质控制流在哪出错修什么归宿
Tool行情终端 / 下单网关确定性计算无——被调用改代码tools/
Skill策略卡 / 风控守则固化的判断标准无——被阅读改手册Skills/ · 域repo .claude/skills/
Agent交易员带判断的执行者（=Claude）模型脑子里改手册 / 加门禁运行时，不是 YAML
Workflow制度化流程确定性控制流，agent 填空剧本/代码里改剧本workflows · pipeline · loamwise(L1)
Team交易台多 agent + 协作结构协作协议里改分工/协议剧本制优先（可审计）

资产硬化梯度（架构总原则）：经验 → Skill(手册) → Workflow(剧本) → Tool(软件)。能力随成熟度下沉——每降一级：确定性↑ 可审计↑ 边际成本↓；边界情况反向上浮给判断，教训写回手册。节制律：从最简形态开始，能用 Tool 别用 Agent，能用单 Agent 别开 Team。

落盘位置：liye_os/_meta/governance/CAPABILITY_MODEL.md（新，与 SKILL_PLACEMENT.md 配对：那个管"放哪"，这个管"是什么"），并植入 AGE CLAUDE.md 顶部。
