# Mission Constraints

## Boundaries
- Stay within project scope
- Follow existing code patterns
- Respect budget limits

## Forbidden Actions
- No destructive operations without explicit approval
- No external API calls without budget consideration
- No modifications to critical system files

## Approval Rules
- All file modifications require review
- External network calls require justification
- Database changes require explicit confirmation

## Security
- No sensitive data in outputs
- No credentials in evidence
- Follow principle of least privilege
