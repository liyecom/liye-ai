# LiYe OS Adopters

> **Purpose**: Track who depends on LiYe OS to manage breaking change impact.

---

## Why Register?

1. **Breaking Change Notifications**: Get notified before Frozen/Stable components change
2. **Influence Evolution**: Your use case shapes our priorities
3. **Risk Assessment**: Help us understand the impact radius of changes

## Private Adopters

If you prefer not to be listed publicly, you can still register anonymously to receive change notifications.

### Anonymous Registration

**Option 1**: Open a [private GitHub Discussion](https://github.com/liyecom/liye_os/discussions) with:
- Adoption path (Blueprint / Governance Stack / Minimal Runtime)
- Which Frozen/Stable components you depend on
- Preferred notification method (GitHub notifications / email)

**Option 2**: Email `liye@liyecom.com` with subject line `[ADOPTER REGISTRATION]`

We will:
- Never disclose your identity
- Notify you before any breaking changes to components you depend on
- Remove you upon request

## Stability Commitment

We commit to:

1. **30-day notice** before any Frozen component changes
2. **14-day notice** before any Stable component changes
3. **Migration guides** for all breaking changes
4. **Direct outreach** to registered adopters

See [ARCHITECTURE_CONTRACT.md](docs/architecture/ARCHITECTURE_CONTRACT.md) for the full stability contract.
