# LiYe AI Glossaries

Terminology and concept definitions for the LiYe AI ecosystem.

## Directory Structure

```
Glossaries/
├── README.md              # This file
├── architecture.md        # Architecture terminology
├── agents.md              # Agent-related terms
└── workflows.md           # Workflow terminology
```

## Purpose

This glossary ensures consistent terminology across:
- Documentation
- Code comments
- Agent communications
- User interfaces

## How to Use

1. **Reference**: Link to glossary entries when using specialized terms
2. **Contribute**: Add new terms as they emerge
3. **Validate**: Check definitions match implementation

## Quick Reference

### Core Concepts

| Term | Definition |
|------|------------|
| **Agent** | An autonomous unit with Persona + Skills + Runtime |
| **Crew** | A coordinated team of agents |
| **Skill** | A discrete capability an agent can use |
| **Domain** | A vertical application area (e.g., amazon-growth) |

### Architecture Layers

| Layer | Responsibility |
|-------|----------------|
| **Method** | WHY - Declarations and definitions |
| **Runtime** | HOW - Execution engine |
| **Skill** | WHAT - Capabilities |
| **Domain** | WHERE - Application context |

## Contributing

When adding new terms:
